# Единая система плавности сайта «Моя Стоматология»

**Дата:** 2026-05-21
**Статус:** утверждён дизайн, готов к плану реализации

## Цель

Добавить согласованную «плавность» по всему сайту: появление секций при
прокрутке, мягкое открытие/закрытие модалок и меню, переходы между
страницами, hover-микроанимации. Характер — сдержанный и премиальный:
короткие мягкие анимации, которые ощущаются, но не отвлекают (уместно для
медицинского сайта, внушает доверие).

## Подход

CSS-переходы и анимации + крошечный хук на `IntersectionObserver`. Без
новых зависимостей. Кодовая база уже целиком на CSS-переходах (base-ui +
`tw-animate-css`), easing-токены частично заведены — подход ложится
естественно. Анимированные секции остаются серверными компонентами,
кроме одной тонкой клиентской обёртки `<Reveal>`.

## Текущее состояние (отправная точка)

- Модалки (`components/ui/dialog.tsx`) — анимируются (`fade-in`, `zoom-in-95`),
  но резко: `duration-100`.
- Бургер-меню (`components/ui/sheet.tsx`) — выезжает за ~200мс через
  base-ui `data-starting-style` / `data-ending-style`.
- Аккордеон FAQ (`components/ui/accordion.tsx`) — раскрывается по высоте
  (`animate-accordion-down` / `-up`).
- `app/globals.css` содержит готовый класс `.reveal` + keyframe `revealUp`,
  **но он не подключён ни к одному блоку** — секции при скролле не появляются.
- Easing-токены `--ease-out-expo`, `--ease-out-quart` уже определены в `@theme`.
- `prefers-reduced-motion: reduce` уже уважается для `.reveal` и smooth-scroll.

## Дизайн

### 1. Токены движения

В `app/globals.css` (`@theme`) добавить единый набор длительностей, на
который ссылаются все анимации:

- `--duration-fast: 150ms` — hover, смена цвета
- `--duration-base: 250ms` — модалки, аккордеон, меню
- `--duration-slow: 400ms` — появление секций, переход страниц

Easing входа — существующий `--ease-out-quart` (`cubic-bezier(0.25,1,0.5,1)`).
Дистанция сдвига reveal — 16px (уже в `.reveal`). Цель: тайминги
согласованы и меняются в одном месте.

### 2. Появление секций при скролле

Новый клиентский компонент `components/motion/Reveal.tsx`:

- Тонкая обёртка-`<div>`, по умолчанию рендерит контент в скрытом
  состоянии (`opacity: 0; transform: translateY(16px)`).
- На монтировании создаёт `IntersectionObserver`; когда элемент входит в
  зону видимости (порог ~0.12, срабатывает один раз) — добавляет класс
  `.reveal` → запускается keyframe `revealUp` (fade + подъём).
- Проп `delay?: number` — задержка анимации (`animation-delay`), для
  каскада карточек в сетке: индекс × 80мс.
- Опциональный проп `as` для смены тега-обёртки не требуется в MVP —
  используем `<div>`; если обёртка ломает раскладку секции, у `<Reveal>`
  есть проп `className` для контроля.
- При `prefers-reduced-motion: reduce` — контент сразу виден (уже
  предусмотрено в CSS-правиле `.reveal`).

Применение: оборачивать **крупные секции**, не каждый элемент. Главная
страница (секции в `components/home/*`) и внутренние страницы (`app/*`):
блоки услуг, врачей, FAQ, отзывов, преимуществ и т.п. Карточки внутри
сеток — с каскадной задержкой через `delay`.

### 3. Тюнинг модалок / меню / аккордеона

- `dialog.tsx`: `duration-100` → `duration-base` (250мс); zoom деликатнее
  (`zoom-in-95` → `zoom-in-98`, `zoom-out-95` → `zoom-out-98`); бэкдроп
  fade синхронно той же длительностью; мягкий easing.
- `sheet.tsx`: длительность привести к `--duration-base`, easing
  согласовать с токеном.
- `accordion.tsx`: длительность раскрытия по высоте и поворот шеврона —
  на `--duration-base` + `--ease-out-quart`.

Цель — единое ощущение открытия/закрытия по всему сайту.

### 4. Переходы между страницами

Новый `app/template.tsx` — Next.js пере-монтирует `template.tsx` при
каждой навигации. Оборачивает `children` в элемент с CSS-классом,
запускающим на монтировании keyframe «fade + лёгкий подъём»
длительностью `--duration-slow` (400мс). Чистый CSS, без библиотек.

Ограничение подхода: анимируется только **вход** на страницу, без
анимации «ухода» — принятый компромисс CSS-подхода.

При `prefers-reduced-motion: reduce` — без анимации.

### 5. Hover-микроанимации

- Карточки (услуги, врачи и аналогичные в `components/ui/card.tsx` и
  местах использования): на наведение — плавный подъём `translateY(-3px)`
  + смена тени `--shadow-soft` → `--shadow-elevated`, длительность
  `--duration-fast`.
- Кнопки и ссылки: убедиться, что присутствует `transition` для цвета/фона,
  длительность `--duration-fast`, мягкий easing.
- Использовать уже заведённые shadow-токены (`--shadow-soft`,
  `--shadow-elevated`, `--shadow-luxe`).

### 6. Доступность и единообразие

- Всё новое движение уважает `@media (prefers-reduced-motion: reduce)`:
  трансформации и анимации opacity отключаются, контент виден сразу.
- Для `.reveal` добавить `<noscript>`-фолбэк в `app/layout.tsx` (`<head>`):
  `<noscript><style>.reveal{opacity:1;transform:none;animation:none}</style></noscript>`
  — при отключённом JS секции не остаются скрытыми.
- Единый набор токенов длительности → согласованные тайминги везде.

## Затрагиваемые файлы

- `app/globals.css` — токены движения, уточнение `.reveal`, keyframe
  перехода страниц
- `components/motion/Reveal.tsx` — **новый**
- `app/template.tsx` — **новый**
- `app/layout.tsx` — `<noscript>`-фолбэк для `.reveal`
- `components/ui/dialog.tsx`, `sheet.tsx`, `accordion.tsx` — тюнинг
  длительностей/easing
- `components/ui/card.tsx` (+ места использования карточек) — hover-подъём
- Обёртки секций в `<Reveal>` по страницам `app/*` и `components/home/*`

## Тестирование

- Playwright: при прокрутке к секции на ней появляется класс `.reveal`.
- Playwright + эмуляция `prefers-reduced-motion: reduce`: контент секций
  виден сразу, без скрытого состояния.
- Playwright: у модалки при открытии присутствует переход (не мгновенный).
- Существующие unit/e2e-тесты не ломаются.

## Вне рамок (YAGNI)

- Библиотека анимаций (`motion` / Framer Motion) — не добавляется.
- Анимация «ухода» страницы при навигации.
- Параллакс, скролл-привязанные (scroll-linked) анимации, сложные
  оркестрации.
